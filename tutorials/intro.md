# 教程专栏

系统性技术教程，帮助你从零开始掌握核心技能。

## 定位

**一句话**：某个技术领域的系统学习路径

**回答的问题**：如何学会这项技术？

## 与其他板块的区别

| 与...的区别 | 教程专栏 | 项目札记 | 技术随笔 |
|------------|---------|---------|---------|
| 核心 | **学习路径** | 开发过程记录 | 技术原理讲解 |
| 受众 | 从零开始的学习者 | 有项目需求的开发者 | 需要查资料的开发者 |
| 组织方式 | 按知识体系递进 | 按项目/时间 | 按知识点独立 |
| 目的 | 学会 | 复用经验 | 解决特定问题 |

**简言之**：

- 教程专栏是"**如何学会**"
- 项目札记是"**怎么做**"
- 技术随笔是"**是什么**"

## 教程体系

```
教程专栏
├── Linux 基础        # Linux 系统入门
│   ├── 系统启动流程
│   ├── 常用命令
│   └── Shell 脚本
├── Web 开发          # 现代 Web 开发入门
│   ├── 前端基础
│   ├── 后端框架
│   └── 部署运维
└── (更多教程规划中)
```

## 教程特点

| 特点 | 说明 |
|------|------|
| **系统性** | 从基础概念到实践应用，循序渐进 |
| **可操作性** | 每一步都有操作指引，可动手实践 |
| **配图丰富** | 关键步骤配有截图或示意图 |
| **配套练习** | 章节末有练习题巩固所学 |

## 写作风格

- 假设读者零基础，不过多依赖前置知识
- 先讲"是什么"和"为什么"，再讲"怎么做"
- 避免过多术语堆砌，必要时解释术语
- 操作步骤清晰明确
