# 技术随笔

独立于项目的技术知识点沉淀。无论你正在做什么项目，这些内容都可能有参考价值。

## 定位

**一句话**：某个技术的深入讲解

**回答的问题**：这个技术是什么？怎么用？有什么坑？

## 内容特点

| 特点 | 说明 |
|------|------|
| **通用性** | 不绑定特定项目，任何开发场景都可能用到 |
| **系统性** | 对某个知识点进行深入讲解，不止于"踩坑记录" |
| **可查阅** | 写成类似博客文章的形式，方便日后回顾 |

## 与其他板块的区别

| 与...的区别 | 技术随笔 | 项目札记 | 教程专栏 |
|------------|---------|---------|---------|
| 核心 | 围绕特定**技术** | 围绕特定**项目** | 围绕**知识体系** |
| 组织方式 | 按知识点 | 按项目/时间 | 按学习路径 |
| 背景依赖 | 独立可读 | 需要项目上下文 | 从零开始 |

**简言之**：

- 技术随笔是"**这个技术**是什么、怎么用"
- 项目札记是"**我在这个项目里**做了什么"
- 教程专栏是"**如何学会**这项技术"

## 涵盖方向

| 方向 | 示例主题 |
|------|---------|
| 前端开发 | React、Vue、TypeScript、Webpack |
| 后端开发 | Node.js、Python、数据库、API 设计 |
| DevOps | Docker、K8s、CI/CD、Linux 系统 |
| 开发工具 | Git、编辑器、调试技巧 |

## 写作风格

- 独立成篇，不依赖其他文档的前置知识
- 从概念到实践，先讲原理再给例子
- 包含常见错误和避坑指南
- 适合作为日后查阅的参考资料
