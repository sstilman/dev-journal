## 核心区别

### 项目札记 vs 技术随笔

这是最容易混淆的两个板块：

| 项目札记 | 技术随笔 |
|---------|---------|
| 围绕特定项目展开 | 围绕特定技术展开 |
| 有项目上下文依赖 | 独立成篇，任何项目都能用 |
| 记录过程：搭建→调试→问题解决 | 讲解原理：概念→语法→应用场景 |

**例子**：

- 项目札记：*"我在 Dev Journal 项目中配置 Docusaurus 侧边栏时遇到的问题"*
- 技术随笔：*"Docusaurus 侧边栏配置的完整指南"*

### 项目札记 vs 教程专栏

| 项目札记 | 教程专栏 |
|---------|---------|
| 记录自己做项目的过程 | 系统性讲解某个技术 |
| 偏向经验分享 | 偏向知识传授 |
| 有个人项目背景 | 无项目背景，从零开始 |

**例子**：

- 项目札记：*"STM32MP1 项目开发日志"*（我做了什么，遇到什么坑）
- 教程专栏：*"STM32 开发入门教程"*（如何学习 STM32）

## 博客是什么

博客与其他板块的核心区别：**其他板块是"写给别人看"，博客是"记录自己"**。

| 板块 | 核心 | 写作心态 |
|------|------|---------|
| 项目札记 | 我做了什么 | 要讲清楚、有条理 |
| 技术随笔 | 这个技术是什么 | 要讲透彻、可查阅 |
| 资源归档 | 去哪找 | 要列清单、可索引 |
| **博客** | 我在想什么 | 自由、随意、真实 |

→ [博客详情](../blog)

## 写作原则

1. **原创优先** — 内容基于实践，避免纯翻译
2. **可复现** — 操作步骤详细，读者能跟着做
3. **注明来源** — 参考资料链接清晰
4. **持续更新** — 发现错误及时修正
